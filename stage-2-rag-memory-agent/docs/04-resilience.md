# 小目标 4：处理工具失败、空结果、重复调用和幻觉引用

## 工具失败

- 工具不把可预期错误直接抛到 Agent Loop。
- `ToolResponse.error` 包含稳定 `error_code` 和可读摘要。
- 注册表将未知工具、超时、异常、错误返回类型统一归一化。
- 模型可以修改参数、选择其他工具或说明限制。

## 空结果

- 空结果使用独立 `status="empty"`，不等同于成功证据，也不伪装为异常。
- `rag_search`、`web_search`、`file_search`、数据库查询和记忆召回均显式处理空结果。
- empty response 没有来源，不会进入证据账本。

## 重复调用

- 工具名和规范化 JSON 参数计算 SHA-256 指纹。
- 每次 run 独立记录调用次数。
- 完全相同的第二次调用不再执行，返回 `REPEATED_TOOL_CALL`。
- 参数发生实质变化时允许重试。
- 最大步骤数和总超时提供最终停止条件。

## 幻觉引用

- 模型无法直接创建来源；只有工具的 `EvidenceSource` 能进入账本。
- 账本只分配当前 run 的 `S1`、`S2`。
- 最终解析要求 `citations` 为字符串数组。
- validator 拒绝未知编号、正文/数组不一致、有证据却无引用。
- 会话历史中的旧编号被明确标记为本轮无效。

## 其他防护

- 工具结果标记为“不可信数据而非指令”。
- LLM、工具和 Agent 分别设置超时。
- 短期上下文有字符预算，避免无限膨胀。
- CLI 输出关键事件，不输出 API Key。

## 对应代码

- `models.py`：三态 ToolResponse。
- `tools/registry.py`：超时与异常归一化。
- `agent/loop.py`：重复调用、步骤、总时限。
- `evidence/validator.py`：引用校验。
- `memory/short_term.py`：上下文预算。

## 验收标准

- [x] error 与 empty 有不同状态和处理语义。
- [x] 相同工具参数不会无限执行。
- [x] 模型不能引用账本之外的编号。
- [x] 步数、总时限和单工具时限均存在。
- [x] 工具异常不会破坏统一结果协议。
- [ ] 用故障注入测试所有错误码和恢复路径。
