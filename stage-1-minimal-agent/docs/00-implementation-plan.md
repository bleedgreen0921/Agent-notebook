# Stage 1 总体实施计划（Python）

## 阶段目标

构建一个边界完整的最小 Agent：接收用户任务，由 LLM 决定直接回答或调用工具，执行工具并回填结果，最终在有限步骤和有限时间内返回答案。

本阶段只研究 Agent Loop 的核心控制流，不引入 RAG、数据库、长期记忆、多 Agent 或浏览器自动化。

## 技术选择

- 语言：Python 3.11+，使用类型标注辅助阅读。
- 数据结构：使用 `dataclass` 表示消息、配置、决策、事件和执行结果。
- 接口抽象：使用 `typing.Protocol` 定义 LLM 和工具接口。
- HTTP：使用标准库 `urllib.request` 调用 `/chat/completions`。
- JSON：使用标准库 `json` 编码请求、解析模型输出。
- 工具执行：使用 `concurrent.futures.ThreadPoolExecutor` 提供限时等待与协作式取消。
- 依赖策略：运行时无第三方依赖。

第一阶段使用自定义 JSON 协议，是为了清楚展示“约束输出、解析决策、执行工具、回填结果”四个环节。后续可以替换 LLM 适配器接入原生 function calling，而不需要重写工具本身。

## 六个小目标

### 1. 普通 LLM 对话

1. 用 `ChatMessage` 表示三种角色消息。
2. 用 `LLMClient` Protocol 隔离模型厂商。
3. 实现 HTTP 请求、认证、超时、响应大小和字段校验。
4. 用 `SimpleChatSession` 保存多轮上下文。
5. 在 CLI 中提供 `--chat` 模式。

验收：非空消息能组成合法请求并得到非空文本；失败请求不会污染历史。

### 2. 结构化 JSON 输出

1. 定义 `tool_call` 与 `final` 两种互斥决策。
2. 从工具注册表动态生成工具说明。
3. 在系统提示词中写明 JSON 结构、单工具限制和失败策略。
4. 使用低温度降低格式波动。
5. 格式错误时把具体问题反馈给模型修正。

验收：每次模型响应都必须转换为工具调用或最终答案，否则不得继续执行。

### 3. 工具定义

1. 用 `Tool` Protocol 定义名称、描述、参数 Schema 和执行函数。
2. 实现工具注册表，拒绝非法名称和重复注册。
3. 实现不调用 `eval` 的四则运算解析器。
4. 实现限制目录、大小、文件类型和 UTF-8 编码的文件读取。
5. 统一记录工具成功、失败和耗时。

验收：工具可以被动态展示和统一调用，不可信参数在具体工具内部再次校验。

### 4. tool call 解析

1. 去除首尾空白和可选的完整 Markdown JSON 围栏。
2. 用 `json.loads()` 解析外部字符串。
3. 使用 `isinstance()` 逐字段验证根对象、类型、名称和参数。
4. 返回 `FinalDecision` 或 `ToolCallDecision`。
5. 非法结果抛出 `ProtocolError`。

验收：任何非法 JSON 或错误字段都不会触发工具。

### 5. 工具执行与结果回填

1. 初始化系统消息和用户任务。
2. 每一步请求一次模型并保存原始输出。
3. 解析决策；`final` 直接结束。
4. `tool_call` 通过注册表执行。
5. 把成功或失败结果序列化并回填给模型。
6. 返回答案、步骤数和不可变消息快照。

验收：任务能经历零次或多次工具调用后得到最终答案，工具失败也能交给模型处理。

### 6. 步数、超时与错误处理

1. 限制 Agent 最大模型决策次数。
2. 分别配置单次 LLM、单次工具和 Agent 总超时。
3. 总剩余时间进一步限制每次 LLM 和工具调用。
4. 区分配置、LLM、协议、工具和 Agent 限制错误。
5. 输出不含密钥的关键 trace。

验收：Agent 不会无限循环，各类失败具有明确行为和错误文本。

## 验收矩阵

| 场景 | 预期行为 |
| --- | --- |
| 普通问题 | 模型返回 `final`，循环一步结束 |
| 计算问题 | 调用 `calculator`，观察结果后返回 `final` |
| 文件问题 | `read_file` 只读取工作区内的小型 UTF-8 文件 |
| 非法模型 JSON | 记录协议错误并要求模型修正 |
| LLM 网络或响应错误 | 抛出 `LLMError` 并终止 |
| 工具参数错误 | 形成失败结果，回填给模型 |
| 工具超时 | 设置取消事件并回填超时结果 |
| 达到最大步骤 | 抛出 `MAX_STEPS` |
| 超过总时限 | 抛出 `AGENT_TIMEOUT` |
